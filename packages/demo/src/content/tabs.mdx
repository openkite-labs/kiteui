# Tabs

Tab components for organizing content into separate views that users can switch between.

## Basic Tabs

Simple tab interface with multiple panels:

<div className="my-6">
  <Tab defaultValue="overview">
    <Tab.List>
      <Tab.Trigger value="overview">Overview</Tab.Trigger>
      <Tab.Trigger value="features">Features</Tab.Trigger>
      <Tab.Trigger value="pricing">Pricing</Tab.Trigger>
    </Tab.List>
    
    <Tab.Content value="overview" className="mt-4 p-4 border border-gray-200 rounded-lg">
      <h3 className="font-semibold mb-2">Product Overview</h3>
      <p className="text-gray-600">
        This is the overview section where you can provide general information about your product or service.
      </p>
    </Tab.Content>
    
    <Tab.Content value="features" className="mt-4 p-4 border border-gray-200 rounded-lg">
      <h3 className="font-semibold mb-2">Key Features</h3>
      <ul className="list-disc list-inside text-gray-600 space-y-1">
        <li>Feature one with detailed description</li>
        <li>Feature two that adds value</li>
        <li>Feature three for enhanced functionality</li>
      </ul>
    </Tab.Content>
    
    <Tab.Content value="pricing" className="mt-4 p-4 border border-gray-200 rounded-lg">
      <h3 className="font-semibold mb-2">Pricing Plans</h3>
      <p className="text-gray-600">
        Choose from our flexible pricing options designed to scale with your needs.
      </p>
    </Tab.Content>
  </Tab>
</div>

```jsx
import { Tab } from '@openkite/kite-ui'

<Tab defaultValue="overview">
  <Tab.List>
    <Tab.Trigger value="overview">Overview</Tab.Trigger>
    <Tab.Trigger value="features">Features</Tab.Trigger>
    <Tab.Trigger value="pricing">Pricing</Tab.Trigger>
  </Tab.List>
  
  <Tab.Content value="overview">
    Overview content here...
  </Tab.Content>
  
  <Tab.Content value="features">
    Features content here...
  </Tab.Content>
  
  <Tab.Content value="pricing">
    Pricing content here...
  </Tab.Content>
</Tab>
```

## Documentation Tabs

Perfect for organizing documentation or code examples:

<div className="my-6">
  <Tab defaultValue="react">
    <Tab.List>
      <Tab.Trigger value="react">React</Tab.Trigger>
      <Tab.Trigger value="vue">Vue</Tab.Trigger>
      <Tab.Trigger value="angular">Angular</Tab.Trigger>
    </Tab.List>
    
    <Tab.Content value="react" className="mt-4">
      <div className="bg-gray-900 text-gray-100 p-4 rounded-lg">
        <pre><code>{`import { Button } from '@openkite/kite-ui'

function App() {
  return (
    <Button variant="primary">
      Click me
    </Button>
  )
}`}</code></pre>
      </div>
    </Tab.Content>
    
    <Tab.Content value="vue" className="mt-4">
      <div className="bg-gray-900 text-gray-100 p-4 rounded-lg">
        <pre><code>{`<template>
  <KiteButton variant="primary">
    Click me
  </KiteButton>
</template>

<script>
import { KiteButton } from '@openkite/kite-ui-vue'

export default {
  components: { KiteButton }
}
</script>`}</code></pre>
      </div>
    </Tab.Content>
    
    <Tab.Content value="angular" className="mt-4">
      <div className="bg-gray-900 text-gray-100 p-4 rounded-lg">
        <pre><code>{`import { Component } from '@angular/core'

@Component({
  selector: 'app-example',
  template: \`
    <kite-button variant="primary">
      Click me
    </kite-button>
  \`
})
export class ExampleComponent {}`}</code></pre>
      </div>
    </Tab.Content>
  </Tab>
</div>

## Best Practices

- Use clear, descriptive labels for tab triggers
- Keep the number of tabs manageable (typically 3-7)
- Ensure content is logically grouped
- Consider mobile responsiveness for tab layouts
- Use consistent content structure across tabs
- Provide visual feedback for the active tab
- Make tabs keyboard accessible
