# Inputs

Form input components with various types and states for collecting user data.

## Basic Input

The default input component for text entry:

<div className="max-w-md space-y-4 my-6">
  <Input placeholder="Enter your text here" />
</div>

```jsx
import { Input } from '@openkite/kite-ui'

<Input placeholder="Enter your text here" />
```

## Input Types

Supports all standard HTML input types:

<div className="max-w-md space-y-4 my-6">
  <Input placeholder="Default text input" />
  <Input placeholder="Email address" type="email" />
  <Input placeholder="Password" type="password" />
  <Input placeholder="Phone number" type="tel" />
  <Input placeholder="Website URL" type="url" />
  <Input placeholder="Search query" type="search" />
</div>

```jsx
<Input placeholder="Email address" type="email" />
<Input placeholder="Password" type="password" />
<Input placeholder="Phone number" type="tel" />
```

## Accessibility

- All inputs support proper ARIA attributes
- Keyboard navigation works as expected
- Screen readers can properly identify input types
- Focus states are clearly visible

## Best Practices

- Always provide clear placeholder text
- Use appropriate input types for better mobile experience
- Provide validation feedback when needed
- Group related inputs logically
- Consider using labels for better accessibility
